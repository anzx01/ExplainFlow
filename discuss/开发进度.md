## 本轮修复

### 1. 去掉纸片感和底影

- 普通白板场景不再默认生成 raster/reference 图片。
- 渲染端新增硬校验，拦截内层 `paper/card/panel/surface`、阴影、滤镜、渐变和淡色纸面。
- 代码生成提示明确要求所有图形直接画在白板上，不允许再包一层白色矩形或卡片。
- Planner 的 fallback 组件移除了 `washD`、`boardShadow`、`textShadow` 等会造成底影的写法。

### 2. 提升通用题材的生动度

- 不再按 MOS/FinFET 等具体词汇特殊处理，统一当作通用讲解题材。
- “通用问题解决框架”固定为 5 个场景：
  1. 全局地图
  2. 结构拆解
  3. 优先取舍
  4. 目标路径
  5. 反馈闭环
- 禁止 coverage 逻辑继续追加“过程拆解/总结”导致重复收尾。
- 把裸勾选符号改成更像老师随手写下的提示词：`有谱`、`别慌`、`下一步`。
- Prompt 要求使用轻松、具体、有课堂感的类比和 doodle，避免空泛标题堆叠。

### 3. 中文和异常文本保护

- API 层增加乱码文本检测。
- 渲染服务增加 Storyboard/job 文本的修复、净化和兜底。
- LLM 输出漂移或文本损坏时，优先回退到稳定的本地通用结构。

### 4. 仓库清理

- 删除 `.tmp` 下的临时日志、e2e JSON、抽帧图片和 trace 文件。
- 从 git 索引移除 `.tmp` 临时产物。
- `.gitignore` 新增 `.tmp/`，后续本地调试文件不会再混进仓库。
- 重写乱码的 `README.md` 和本文件为 UTF-8 中文。

## 仍需关注

- 抽帧级视觉质检还需要继续跑更多题材，确认所有场景都没有纸片感底影。
- 幽默感和生动度目前主要靠 prompt 与固定场景策略，后续可以沉淀一套更稳定的 doodle/比喻库。
- 前端可补一个“最近输出/打开视频”的入口，减少手动找 `outputs` 文件的成本。
- 如果要长期保留 e2e 证据，建议输出到专门的 ignored 目录并只提交摘要报告。
