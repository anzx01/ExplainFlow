## 当前技术方法

### 1. 总体生成链路

```text
用户主题/参考图
  -> Storyboard
  -> TTS
  -> LLM/本地编译器生成自包含 Remotion TSX
  -> Remotion bundle
  -> renderMedia 输出 MP4
```

关键约束：

- Remotion 动画必须由 `useCurrentFrame()`、`interpolate()` 和 `Sequence` 驱动。
- 生成代码必须导出 `GeneratedVideo`。
- 只允许导入 `react` 和 `remotion`。
- 只允许受控 `staticFile()`：手部图片和 job-local 参考图资产。
- 禁止任意 `fs`、网络 fetch、模板库、组件库、CSS animation、SVG animate。

### 2. LLM 连通性与失败保护

- 渲染前通过 `/health/llm?force=true` 检查 DeepSeek/OpenAI-compatible 接口。
- 如果 LLM 不可用，立即停止，不继续 TTS、codegen、bundle 或 render。
- `OPENAI_BASE_URL`、`LLM_MODEL`、`CODER_MODEL`、`OPENAI_API_KEY` 可放在 `.env` 或进程环境变量中；日志不回显 key。

### 3. 原图逐笔揭示方案

核心思想：保留原图线稿像素质量，用路径遮罩逐步揭示原图，而不是把图重新矢量化后重画。

实现步骤：

1. `sharp` 读取本地参考图，使用宽容模式处理截断 JPEG。
2. 去除黄色/纸张背景，将深色线稿转成透明 PNG。
3. 从 mask 中提取深色区域，清理小噪点。
4. 使用骨架化算法得到笔画中心线。
5. 按端点、交叉点、闭环和空间顺序切分路径。
6. 控制路径数量约 120-150 条，避免太碎导致手乱跳。
7. 为每条路径估算 `revealWidth`，遮罩宽度覆盖原图笔画但尽量不误揭示相邻元素。
8. Remotion 中用 SVG `<mask>` 和 `strokeDashoffset` 逐条打开透明线稿图。
9. `HandPen` 使用同一批 `drawOps` 和 `pointOnPolyline()`，让笔尖沿当前笔画移动。
10. 末尾淡出 masked SVG 图层，并淡入 HTML `<Img>` 最终图层，避免透明 PNG 在 SVG 中出现黑底或双重加深。

效果收益：

- 中文不再是粗空心字。
- 圆圈、箭头、连接线保留原图锐利边缘。
- 首帧几乎空白，中间帧逐步出现，末帧接近原图。
- 画笔不是简单滑动，而是沿元素中心线移动。

### 4. 编码与画质

- Remotion 先输出 PNG 帧，避免中间帧压缩损失。
- MP4 使用 H.264，默认 CRF 8。
- 默认 `pixelFormat: "yuv444p"`，比 `yuv420p` 更适合高对比黑白线稿。
- 参考图资产默认生成到最终展示尺寸附近，减少浏览器二次缩放带来的边缘偏差。

### 5. 通用绘制策略

不再靠枚举 MOS、FinFET、医学、机械等具体场景来决定画法，而是抽象为老师板书策略：

- `trace`：结构简单、元素少、可分解为少量步骤，适合从空白白板逐笔揭示。
- `direct`：图像密集、标签很多、多层结构、实物/参考图/成品图，直接呈现主体，避免手部在大量路径间乱跳。
- `hybrid`：复杂主体直接呈现，手部只负责写局部箭头、圈、下划线、重点词和 callout。

新增课堂表达字段：

- `board_mode=whiteboard`：浅灰白板，适合机制图、流程图、结构图和状态对比。
- `board_mode=clean_canvas`：干净浅色画布，适合营销/产品/概念介绍中的彩色 doodle 主体。
- `board_mode=reference`：复杂参考图、三维结构、医学/机械/电路图，主体直接或混合呈现。
- `board_mode=chalkboard`：深色黑板，适合数学推导、公式证明、解题过程，通常 `hand_usage=none`。
- `hand_usage=trace|annotate|none`：分别表示逐笔写画、只做局部标注、无手逐步显现。
- `visual_style=teacher_whiteboard|marketing_doodle|math_chalkboard|technical_reference`：表达课堂视觉语法，不引入模板库。

决策信号：

- Storyboard 输出 `render_strategy` 和 `visual_complexity`，表达内容层面的推荐。
- Storyboard 也输出 `board_mode`、`hand_usage`、`visual_style`，表达课堂呈现方式。
- 渲染端再结合标签数量、visual beat 数量、线稿骨架像素数、mask 覆盖率和路径数量校准。
- 规则目标是通用可迁移：简单图像参考视频一样实时写画，复杂图像优秀老师一样先给清楚主体再讲重点。

### 6. 教学关系视觉语法

最新抽象重点：不要围绕某个具体题材词继续打补丁。系统先判断“这一场要讲清什么关系”，再选择画面结构、绘制节奏和图像策略。

通用 `diagram_plan.kind`：

- `overview_map`：主题下有多个对象、单元或阶段，先画全局地图。
- `comparison`：前后、开关、A/B、旧新、好坏、状态变化，用双图对比。
- `process`：因果链、机制、流程、变化过程，用原因 -> 过程 -> 结果。
- `structure`：组成、层级、整体和局部，用主体结构图、贴近标签和局部放大。
- `interaction`：对象之间互相影响、沟通、协作、交换、反馈，用节点和双向箭头。
- `tradeoff_matrix`：优先级、取舍、二维判断、分类，用坐标/四象限/2x2 矩阵。
- `goal_path`：目标、路线、里程碑、倒推、成长路径，用起点到终点路径图。
- `cycle`：迭代、复盘、更新、闭环，用环形箭头和 3-5 个节点。
- `formula`：公式含义，用公式、变量 callout 和小示意图。
- `reference_callout`：复杂主体需要保真时直接呈现，再用 2-4 个局部 callout 讲重点。
- `summary`：收束时用少量清单、闭环或框架图，不写长段落。

落地位置：

- `services/api/src/explain/service.py`：Prompt enhancement 要求输出 teaching coverage units，并按教学关系推荐 visual_role。
- `services/api/src/planner/service.py`：Storyboard prompt 要求使用通用 `diagram_plan.kind`；本地 Remotion 兜底绘图按关系选择 overview、comparison、process、interaction、matrix、path、cycle、formula 等图形。
- `apps/render/server.mjs`：Seedream/reference 图候选评分改为按视觉关系、复杂度和策略字段评分，不再按某类主题词加权。

质量判定也随之统一：当视频无趣、单调、音画不合或场景不连续时，优先检查“关系是否抽对、coverage unit 是否漏掉、beat 是否对齐、复杂图是否应该 direct/hybrid”，而不是继续为单个主题写特殊规则。

### 7. 参考样片规则沉淀

已对 `whiteboard-style-1.mp4`、`ads-marketing-demo.mp4`、`iit-math-q5-solution.mp4` 做关键帧抽样观察，并把其 prompt 与排版精髓抽象成通用规则：

- 内容节奏：一屏只推进一个核心想法，逐步补板书，而不是一次性做完整信息图。
- 版式：标题在顶部左侧或顶部居中，蓝色手写；主体图在中部或略偏右，占画面约 45%-65%，保留大面积空白。
- 文本：板书只放短标签和一句结论，标签靠近对象；避免长段落、固定左侧说明栏和 bullet-list 页面。
- 颜色：黑色主体线；蓝色用于标题/控制；红色用于风险、电流或错误；绿色用于有效路径和正确结果；黄色只做短下划线或局部强调。
- 手部：画笔必须出现在当前笔画附近，简单图逐笔 trace；复杂主体直接呈现后，只补少量箭头、圈选、下划线和 callout。
- 图像提示：要求 light grey-white empty whiteboard、black marker line art、sparse blue handwritten labels、large negative space；禁止 poster/card/legend/panel/background wash。
- 营销 doodle：彩色成品对象可以直接出现，手负责写标题、勾选、箭头、下划线和短标注，不强行逐笔描完每个图标。
- 数学黑板：允许无手；深色背景，公式/向量/推导按行显现，白色主式、cyan/green 变量、yellow 结论、pink/red 条件。
- 复杂参考图：先清楚呈现主体，再用 2-4 个局部 callout 讲解重点，避免几百条路径导致手乱跳。

这些规则已落到：

- `services/api/src/explain/models.py` / `service.py`：EnhancedTeachingBrief 增加推荐的 board/hand/style 策略，并沉淀白板、营销 doodle、黑板推导、复杂参考图规则。
- `services/api/src/planner/models.py` / `service.py`：Scene 增加 `board_mode`、`hand_usage`、`visual_style`；Storyboard prompt 和 Remotion codegen prompt 都要求选择场景策略。
- `services/api/src/imagegen/service.py`：Seedream prompt 按白板线稿、营销彩色 doodle、技术参考图、黑板推导切换风格 suffix。
- `apps/render/server.mjs`：复杂图策略判定纳入 board/hand/style；黑板/无手场景跳过 Seedream 白板 tracing；codegen style prompt 支持无手黑板和 direct/annotate。
- `apps/web/src/lib/types.ts`：前端 Scene 类型补充新策略字段。

### 8. 前端进度与预计剩余时间

- 任务阶段：`queued`、`tts`、`imagegen`、`codegen`、`bundling`、`rendering`、`done`。
- 前端根据阶段权重、当前进度、故事板总时长和已耗时估算剩余时间。
- `/storyboard` 渲染按钮和 `/jobs` 任务页都展示预计剩余时间。
